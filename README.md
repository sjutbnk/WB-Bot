# Wildberries Automation Telegram Bots

Система из двух независимых Telegram-ботов с общей базой данных **PostgreSQL** для автоматизации процессов селлера Wildberries:
1. **Бот по рекламе и аналитике (Ad & Funnel Bot)**: Отслеживает воронку продаж, анализирует CTR, CPC, CR по рекламным кампаниям и отдельным артикулам, переписывает описания под SEO и составляет ТЗ для инфографики (с помощью Gemini API), отслеживает цены и остатки конкурентов.
2. **Бот по планированию поставок (Supply Planning Bot)**: Раз в неделю отправляет Excel-отчет с подробным расчетом оборачиваемости и потребностей в поставках с распределением по 5 ключевым региональным складам WB.

---

## 🛠 Технологический стек
* **Язык**: Python 3.11
* **Фреймворк**: [aiogram 3.x](https://github.com/aiogram/aiogram) (асинхронная библиотека для Telegram API)
* **База данных**: PostgreSQL + SQLAlchemy (асинхронный движок asyncpg)
* **Анализ данных**: Pandas, openpyxl (для генерации Excel отчетов)
* **ИИ-ассистент**: Google Gemini API (`google-generativeai`)
* **Планировщик**: APScheduler (для автоматической рассылки утренних и еженедельных отчетов)
* **Контейнеризация**: Docker, Docker Compose

## 🚀 Быстрый старт

### 1. Настройка окружения
Скопируйте файл `.env.example` в `.env`:
```bash
cp .env.example .env
```
Заполните переменные в `.env`:
* `BOT_TOKEN_ADV`: Токен первого бота (Реклама) из `@BotFather`.
* `BOT_TOKEN_SUPPLY`: Токен второго бота (Поставки) из `@BotFather`.
* `WB_API_TOKEN` и `WB_ADV_TOKEN`: Ваши ключи API Wildberries. (Если оставить дефолтными или написать "mock", бот будет отдавать реалистичные демо-данные).
* `GEMINI_API_KEY`: Ваш токен доступа к Gemini API.
* `ADMIN_IDS`: Telegram ID через запятую для пользователей, которым разрешен доступ к боту.

### 2. Запуск через Docker Compose (Рекомендуемый способ)
Соберите образы и запустите контейнеры:
```bash
docker-compose up -d --build
```
Это автоматически запустит:
1. Базу данных PostgreSQL (с сохранением данных в Docker Volume).
2. Контейнер Бота по рекламе (`maria_bot_adv`).
3. Контейнер Бота по поставкам (`maria_bot_supply`).

Проверить логи контейнеров:
```bash
docker-compose logs -f
```

Остановить проект:
```bash
docker-compose down
```

## 🧪 Запуск и тестирование локально (без Docker)

1. Установите виртуальное окружение и зависимости:
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

2. Убедитесь, что у вас запущена локальная база данных PostgreSQL и переменные подключения настроены в `.env`.

3. Запустите Бота по рекламе:
```bash
python main_adv.py
```

4. Запустите Бота по планированию поставок в другом терминале:
```bash
python main_supply.py
```

## 📈 Алгоритмы и логика расчета

### Воронка РК (Бот по рекламе)
Бот считывает просмотры, клики, корзины и заказы по активным кампаниям. На основе CTR, CR в корзину и CR в заказ генерируются автоматические советы:
* **CTR < 2.0%**: Рекомендация переделать главный слайд обложки (можно запустить Gemini для написания ТЗ).
* **CR в корзину < 8.0%**: Проблема с наполнением карточки (непонятные выгоды, нет размерной сетки).
* **CR в заказ < 15.0%**: Товар бросают в корзине (просадка по цене, долгая доставка или плохие отзывы на первой странице).

### Потребность в поставках (Бот по поставкам)
* **Скорость продаж** рассчитывается как максимальная среднесуточная скорость за 14 и 30 дней.
* **Оборачиваемость** рассчитывается как отношение текущего остатка на складах к скорости продаж.
* **Общая потребность** рассчитывается для покрытия спроса на 30 дней с учетом текущих остатков и товаров в пути (к клиенту/от клиента).
* **Региональное распределение**: Рекомендуемый объем поставки распределяется между складами (Коледино, Казань, Электросталь, Краснодар, Екатеринбург) пропорционально доле продаж с этих складов за последние 30 дней.
* В Excel-файле строки с товарами, по которым рассчитана не нулевая потребность в поставке, автоматически окрашиваются в мягкий зеленый цвет.
